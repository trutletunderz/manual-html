# Design Tweak (xFish)

เป้าหมาย: แก้ดีไซน์เล็กๆ (ตำแหน่ง / สี / spacing / layout) แบบ **surgical, deterministic, token-cheap**
อย่าแก้เกินกว่าที่สั่ง อย่า reformat อย่าจัดระเบียบ class ที่ไม่เกี่ยว

---

## 1. Class-diff syntax (รูปแบบ input หลัก)

ผู้ใช้สั่งเป็น `+`/`-` ต่อ selector — apply ตรงๆ ไม่ต้องตีความ

| สัญลักษณ์ | ความหมาย |
|---|---|
| `+class` | เพิ่ม class นี้เข้า element |
| `-class` | ลบ class นี้ออก |
| `+a -b` | ทำพร้อมกันได้ ตามลำดับซ้ายไปขวา |

ตัวอย่าง mapping จาก prose → class-diff (ถ้าผู้ใช้พิมพ์เป็นภาษาพูด ให้แปลงเป็นอันนี้ในใจก่อน):

| ภาษาพูด | class-diff |
|---|---|
| ย้ายไปขวาสุด | `+ml-auto` |
| ชิดบน / ชิดล่างในแนวตั้ง | `+self-start` / `+self-end` |
| เรียงแนวนอนแทนแนวตั้ง ชิดขวา | `-flex-col +flex-row +justify-end` |
| เว้นห่างมากขึ้นระหว่าง item | `-gap-2 +gap-4` |
| พื้นหลังสีหลัก ตัวอักษรขาว | `+bg-primary +text-white` |
| ทำให้กว้างเต็ม | `+w-full` |
| ซ่อนบนมือถือ โชว์บน desktop | `+hidden +md:block` |

ถ้าตำแหน่งบอกเป็น class ยาก (เช่น absolute positioning ซับซ้อน) และผู้ใช้แนบรูปมีลูกศร/กรอบ → อ้างอิงรูปได้ ไม่ต้องเดาพิกัด

---

## 2. กฎ xFish (บังคับ — ห้ามฝ่าฝืน)

1. **Tailwind core utility เท่านั้น** — ห้าม arbitrary value เด็ดขาด
   - ❌ `w-[437px]` `text-[#1a2b3c]` `mt-[13px]`
   - ✅ `w-full` `max-w-md` `text-primary` `mt-3`
   - ถ้าผู้ใช้สั่ง arbitrary value มา → หยุด ทักก่อน เสนอ token/utility ใกล้เคียง อย่าเงียบแล้ว apply
2. **ห้าม custom CSS / `<style>` / inline `style=""`** — ทุกอย่างผ่าน class
3. **ห้าม raw HTML form** (`<form>` `<input>` `<select>` ดิบ) — ใช้ Naive UI component
4. **สีต้องเป็น token** ของ design system ไม่ใช่ hex ดิบ ไม่ใช่ `blue-500` ลอยๆ
   - **ห้ามเดาชื่อ token** — ก่อน flag หรือเสนอชื่อสี ให้อ่านจาก source จริงก่อน (resolve ตามลำดับด้านล่าง) เหมือนกฎ "ห้ามเดา Bruno field ก่อนอ่าน .bru"
   - ถ้า resolve แล้วไม่เจอ token ที่ตรง → บอกตรงๆ ว่าไม่เจอ อย่าแต่งชื่อขึ้นมาเอง
5. แตะเฉพาะ element ที่ระบุใน selector — element อื่นห้ามขยับ

### Token resolution order (อ่านไฟล์แรกที่เจอ แล้วใช้ชื่อจากที่นั่น)
1. `tailwind.config.ts` / `tailwind.config.js` → `theme.extend.colors` (ชื่อ token + alias เช่น `primary`, `surface`)
2. theme override ของ Naive UI (เช่น ที่ตั้ง `commonThemeOverrides` / `themeOverrides` — มัก provide ผ่าน `NConfigProvider`) → สีระดับ component
3. CSS custom properties `--*` ใน global stylesheet / `app.vue` `<style>` หรือ `assets/css/*`
- เจอจากไฟล์ไหน ให้ยึดชื่อจากไฟล์นั้น และอ้างอิง path ที่อ่านมาในบรรทัดเตือน (สั้นๆ) เพื่อให้ตรวจย้อนได้

---

## 3. Naive UI mapping (เตือนถ้าเข้าข่าย)

ถ้า element ที่กำลังแก้เป็น raw HTML ที่ควรเป็น Naive UI component อยู่แล้ว ให้เตือนสั้นๆ 1 บรรทัดก่อน apply (ไม่ต้องบังคับเปลี่ยนถ้าผู้ใช้แค่จะ tweak สี แต่ flag ไว้):

| raw HTML pattern | → Naive UI |
|---|---|
| `<table>` แสดงข้อมูล | `NDataTable` |
| `<button>` | `NButton` |
| `<input>` / `<select>` / `<textarea>` | `NInput` / `NSelect` / `NInput type="textarea"` |
| card container | `NCard` |
| modal / popup | `NModal` |
| dropdown menu | `NDropdown` |
| tag / badge | `NTag` |
| tab bar | `NTabs` |

---

## 4. Output format (สำคัญที่สุดสำหรับประหยัด token)

- **output เป็น unified diff เท่านั้น** — แสดงเฉพาะบรรทัดที่เปลี่ยน พร้อม context 1-2 บรรทัด
- ห้าม print ทั้งไฟล์
- ห้ามเขียนคำอธิบายยาว — ถ้าจำเป็นต้องเตือน (กฎข้อ 2/3) ให้สั้นที่สุด 1 บรรทัด แล้วตามด้วย diff
- ถ้าแก้หลายไฟล์ ให้แยก diff เป็น block ต่อไฟล์ หัวบล็อกเป็น path

ตัวอย่าง output ที่ถูก:
```
~ ลบ flex-col เพื่อเรียงแนวนอน, ขยับการ์ดชิดขวา

--- pages/dashboard/components/StatsCard.vue
   <div class="flex flex-col gap-2">
+  <div class="flex flex-row justify-end gap-4 ml-auto">
```

## 5. กรณีกำกวม

- selector ตรงกับหลาย element → ถามว่าหมายถึงตัวไหน อย่าแก้มั่ว
- class-diff ขัดแย้งกันเอง (เช่น `+flex-col +flex-row`) → ทักก่อน
- ผู้ใช้สั่งเปลี่ยนแบบที่กระทบ component อื่น → flag ผลข้างเคียงสั้นๆ ก่อน apply
เลื่อนดูเนื้อหาเต็มในกล่องได้ · ปุ่ม Copy คัดลอกทั้งไฟล์

04

ติดตั้ง strip hook

วางไฟล์ .mjs นี้ลง .claude/hooks/ ไม่ต้อง chmod (เรียกผ่าน node) ตัวมัน self-contained ไม่มี dependency อ่าน mockup จาก disk → strip → คืนผ่าน updatedToolOutput และ fail-safe: error/JSON เพี้ยน/partial read/ไม่ใช่ .html → ปล่อยผ่านเงียบ ไม่เคยบล็อก Read

⎔ posttooluse-strip-html.mjs → .claude/hooks/ 
#!/usr/bin/env node
/**
 * posttooluse-strip-html.mjs — Claude Code PostToolUse hook (matcher: Read)
 *
 * When Claude reads a large .html mockup, this rewrites the tool output with a
 * stripped version BEFORE it enters Claude's context (uses updatedToolOutput,
 * supported for built-in tools in Claude Code v2.1.121+).
 *
 * Self-contained (no imports, no deps) so it works regardless of cwd.
 *
 * Register in .claude/settings.json:
 *   {
 *     "hooks": {
 *       "PostToolUse": [
 *         {
 *           "matcher": "Read",
 *           "hooks": [
 *             { "type": "command", "command": "node .claude/hooks/posttooluse-strip-html.mjs" }
 *           ]
 *         }
 *       ]
 *     }
 *   }
 *
 * Safety: on ANY uncertainty it passes through untouched (exit 0, no output).
 * It never blocks a read and never throws into Claude Code.
 */

const THRESHOLD_CHARS = 6000; // ~1.5k tokens; below this, not worth stripping

function strip(html) {
  return html
    .replace(/<!--[\s\S]*?-->/g, '')
    .replace(/<script\b[^>]*>[\s\S]*?<\/script>/gi, '')
    .replace(/<style\b[^>]*>[\s\S]*?<\/style>/gi, '')
    .replace(/<noscript\b[^>]*>[\s\S]*?<\/noscript>/gi, '')
    .replace(/<link\b[^>]*>/gi, '')
    .replace(/<meta\b[^>]*>/gi, '')
    .replace(/(<svg\b[^>]*>)[\s\S]*?<\/svg>/gi, '$1</svg>')
    .replace(/data:image\/[a-z.+-]+;base64,[A-Za-z0-9+/=\s]+/gi, 'data:image-stripped')
    .replace(/>\s+</g, '>\n<')
    .replace(/\n{3,}/g, '\n\n')
    .trim();
}

const passThrough = () => process.exit(0); // do nothing, Claude sees original

async function main() {
  // read hook JSON from stdin
  let input = '';
  process.stdin.setEncoding('utf8');
  for await (const chunk of process.stdin) input += chunk;

  let payload;
  try {
    payload = JSON.parse(input);
  } catch {
    return passThrough();
  }

  if (payload.tool_name !== 'Read') return passThrough();

  const ti = payload.tool_input || {};
  const filePath = ti.file_path || '';

  // only html mockups
  if (!/\.html?$/i.test(filePath)) return passThrough();
  // skip partial reads — re-reading the whole file would change what was asked for
  if (ti.offset != null || ti.limit != null) return passThrough();

  // read the clean file from disk (avoids Read's line-number formatting)
  let raw;
  try {
    const fs = await import('node:fs/promises');
    raw = await fs.readFile(filePath, 'utf8');
  } catch {
    return passThrough();
  }

  if (raw.length < THRESHOLD_CHARS) return passThrough();

  const stripped = strip(raw);

  // only intervene if we actually saved something meaningful (>15%)
  if (stripped.length >= raw.length * 0.85) return passThrough();

  const note =
    `<!-- [auto-stripped mockup: ${raw.length}->${stripped.length} chars, ` +
    `~${Math.round((raw.length - stripped.length) / 4)} tokens saved. ` +
    `scripts/styles/meta/svg-paths/base64 removed; DOM + Tailwind classes + text intact] -->\n`;

  const out = {
    hookSpecificOutput: {
      hookEventName: 'PostToolUse',
      updatedToolOutput: note + stripped,
    },
  };
  process.stdout.write(JSON.stringify(out));
  process.exit(0);
}

main().catch(() => passThrough());
self-contained · zero-dependency · removal-only

06

standalone stripper

optional

ตัวเดียวกับ hook แต่เรียกตรงจาก CLI/โค้ดได้ เหมาะเสียบเป็น step ก่อน Opus vision ใน pipeline spec generation (Hermes/Vultr) วางที่ไหนก็ได้ เช่น scripts/strip-mockup.mjs

⎔ strip-mockup.mjs → scripts/ (อิสระ) 
#!/usr/bin/env node
/**
 * strip-mockup.mjs — deterministic HTML mockup noise remover (zero-dependency)
 *
 * Removes the heavy, design-irrelevant parts of an HTML mockup while keeping
 * the DOM structure, Tailwind classes, and text intact — so an LLM gets the
 * design intent at a fraction of the token cost.
 *
 * It only DELETES well-delimited blocks (never re-parses/rebuilds structure),
 * so it is safe and deterministic without a DOM library.
 *
 * Usage (standalone):
 *   node strip-mockup.mjs path/to/mockup.html        # -> stripped HTML to stdout, stats to stderr
 *   cat mockup.html | node strip-mockup.mjs          # -> reads stdin
 *
 * Importable:
 *   import { strip } from './strip-mockup.mjs'
 *   const { html, stats } = strip(rawHtml)
 */

export function strip(html) {
  const before = html.length;

  let out = html
    // HTML comments
    .replace(/<!--[\s\S]*?-->/g, '')
    // <script>...</script>  (with or without attributes / body)
    .replace(/<script\b[^>]*>[\s\S]*?<\/script>/gi, '')
    // <style>...</style>  (Tailwind utilities live in class=, not here)
    .replace(/<style\b[^>]*>[\s\S]*?<\/style>/gi, '')
    // <noscript>...</noscript>
    .replace(/<noscript\b[^>]*>[\s\S]*?<\/noscript>/gi, '')
    // <link ...> and <meta ...>  (preload/charset/og noise — keep <title>)
    .replace(/<link\b[^>]*>/gi, '')
    .replace(/<meta\b[^>]*>/gi, '')
    // SVG icons: keep the opening <svg> tag (its class = sizing/color matters),
    // drop the long path soup inside.
    .replace(/(<svg\b[^>]*>)[\s\S]*?<\/svg>/gi, '$1</svg>')
    // base64 data URIs inside any attribute: keep the attribute, kill the payload
    .replace(/data:image\/[a-z.+-]+;base64,[A-Za-z0-9+/=\s]+/gi, 'data:image-stripped')
    // collapse whitespace between tags
    .replace(/>\s+</g, '>\n<')
    // collapse 3+ blank lines
    .replace(/\n{3,}/g, '\n\n')
    .trim();

  const after = out.length;
  const tokEst = (n) => Math.round(n / 4); // ~4 chars per token, rough
  const stats = {
    beforeChars: before,
    afterChars: after,
    reductionPct: before ? Math.round((1 - after / before) * 100) : 0,
    beforeTokensEst: tokEst(before),
    afterTokensEst: tokEst(after),
    savedTokensEst: tokEst(before) - tokEst(after),
  };
  return { html: out, stats };
}

// ---- CLI entry ----
const isMain = import.meta.url === `file://${process.argv[1]}`;
if (isMain) {
  const readAll = (stream) =>
    new Promise((res) => {
      let d = '';
      stream.setEncoding('utf8');
      stream.on('data', (c) => (d += c));
      stream.on('end', () => res(d));
    });

  const run = async () => {
    const fileArg = process.argv[2];
    let raw;
    if (fileArg) {
      const fs = await import('node:fs/promises');
      raw = await fs.readFile(fileArg, 'utf8');
    } else {
      raw = await readAll(process.stdin);
    }
    const { html, stats } = strip(raw);
    process.stdout.write(html);
    process.stderr.write(
      `\n[strip-mockup] ${stats.beforeChars}→${stats.afterChars} chars ` +
        `(-${stats.reductionPct}%) ~${stats.savedTokensEst} tokens saved\n`
    );
  };
  run().catch((e) => {
    process.stderr.write(`[strip-mockup] error: ${e.message}\n`);
    process.exit(1);
  });
}
