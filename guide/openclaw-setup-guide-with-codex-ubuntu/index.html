4 หลักคิดที่ร้อยเรียงทั้งเล่ม

CONFIG

แก้แล้วต้อง restart

Gateway โหลด config เข้า memory ตอน start — สาเหตุอันดับ 1 ของทุกปัญหา

MODEL

ใช้ให้ตรงงาน

Codex สำหรับ coding + tool-calling · main แพง subagent ถูก คุมต้นทุน

SECURITY

เริ่มแคบแล้วค่อยเปิด

Identity → Scope → Model · สมมติว่าโมเดลถูกหลอกได้ จำกัด blast radius

WHY

เข้าใจ "ทำไม"

รู้กลไกเบื้องหลัง = เดาปัญหาใหม่ออกเอง ไม่ต้องท่องวิธีแก้

Quick Reference — จำขึ้นใจ

ติดตั้งcurl -fsSL https://openclaw.ai/install.sh | bash หรือ npm i -g openclaw@latest
เช็กสุขภาพopenclaw doctor · openclaw gateway status · openclaw logs --follow
เชื่อม Codexopenclaw models auth login --provider openai-codex + model openai/gpt-5.5
Pairingopenclaw pairing list <channel> → openclaw pairing approve <channel> <CODE>
หลังแก้ configopenclaw gateway restart — ทุกครั้ง ไม่มีข้อยกเว้น
ความปลอดภัยopenclaw security audit --deep · ตั้ง dmPolicy: "allowlist"
อัปเดตopenclaw update (detect, ดึงล่าสุด, doctor, restart ให้อัตโนมัติ)
สำรองก่อนแก้ใหญ่openclaw backup create
