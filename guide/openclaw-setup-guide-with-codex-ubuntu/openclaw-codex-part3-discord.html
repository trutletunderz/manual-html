Discord คือ channel ที่ทรงพลังที่สุดของ OpenClaw — แต่ละห้อง (channel) เป็น session แยกของตัวเอง มี context/memory เป็นของตัวเอง รองรับ thread, slash command, voice และ component UI · ที่สำคัญสำหรับสายโค้ด: ผูกห้องเข้ากับ Codex แบบ persistent ให้ทำงานในโปรเจกต์จริงได้

01 สร้าง Application + Bot

แนะนำสร้าง server ส่วนตัวของตัวเองก่อน (เลือก Create My Own → For me and my friends)

1
เปิด Discord Developer Portal → กด New Application ตั้งชื่อ เช่น "OpenClaw"
2
เมนูซ้ายเลือก Bot → ตั้ง Username ตามชื่อ agent ที่อยากให้โชว์
🏠
ยังไม่มี server? สร้างก่อนใน Discord app (ปุ่ม + ด้านซ้ายล่าง → Create My Own → For me and my friends) — แนะนำให้ใช้ server ส่วนตัวที่มีคุณกับ bot

02 เปิด Privileged Gateway Intents

อยู่ในหน้า Bot เลื่อนลงไปหา "Privileged Gateway Intents" — ขั้นนี้ลืมบ่อยที่สุดและทำให้ bot ไม่ตอบ

✓ Message Content Intent — จำเป็น
✓ Server Members Intent — แนะนำ (จำเป็นถ้าใช้ role allowlist / จับคู่ชื่อ→ID)
○ Presence Intent — ไม่บังคับ (ใช้เฉพาะ presence)
🚨
ถ้าลืมเปิด Message Content Intent → bot จะไม่เห็นข้อความเลย และต้อง restart Gateway ทุกครั้งหลังเปลี่ยน intents

03 เอา Bot Token

ยังอยู่ในหน้า Bot

1
เลื่อนขึ้นไปกด Reset Token
2
คัดลอก token ที่ได้เก็บไว้ — นี่คือ Bot Token
ℹ️
ชื่อปุ่มว่า "Reset" แต่ครั้งแรกมันคือการ สร้าง token ครั้งแรก ไม่ได้รีเซ็ตอะไร — ไม่ต้องตกใจ
🔐
Token = ความลับเหมือนรหัสผ่าน · เก็บไว้ในที่ปลอดภัย อย่า commit ขึ้น Git ถ้าหลุดให้ Reset ใหม่ทันที

04 สร้าง Invite URL + เชิญ bot เข้า server

เมนูซ้ายเลือก OAuth2 → เลื่อนไปหา "OAuth2 URL Generator"

เลือก Scopes

✓ bot
✓ applications.commands

เลือก Bot Permissions (ชุดพื้นฐาน)

✓ View Channels
✓ Send Messages
✓ Read Message History
✓ Embed Links
✓ Attach Files
○ Add Reactions (ไม่บังคับ)
○ Send Messages in Threads (ถ้าใช้ thread/forum)
→
คัดลอก URL ที่ generate ด้านล่าง → เปิดในเบราว์เซอร์ → เลือก server ของคุณ → กด Continue / Authorize · เสร็จแล้วจะเห็น bot อยู่ใน server
🛡️
หลักการ least-privilege: เปิดเฉพาะสิทธิ์ที่จำเป็น — อย่าให้ Administrator เพราะถ้า token หลุด ความเสียหายจะจำกัด

05 เอา Server ID + User ID

ต้องเปิด Developer Mode ใน Discord ก่อนถึงจะ copy ID ได้

1
เปิด Developer Mode: User Settings (ฟันเฟือง) → Advanced → เปิด Developer Mode
2
คลิกขวาที่ ไอคอน server ด้านซ้าย → Copy Server ID
3
คลิกขวาที่ avatar ของตัวเอง → Copy User ID
4
เปิดให้ bot DM คุณได้ (เพื่อทำ pairing): คลิกขวา ไอคอน server → Privacy Settings → เปิด Direct Messages
📝
เก็บ 3 อย่างนี้ไว้ด้วยกัน: Bot Token + Server ID + User ID — ใช้ในขั้นถัดไปทั้งหมด

06 ใส่ token ให้ OpenClaw + สตาร์ท

วิธีแนะนำคือเก็บ token ใน env แล้วชี้ config ไปหา (ปลอดภัยกว่าเขียน token ลงไฟล์)

ตั้ง env + เขียน config (วิธีทางการ)

bash — env + config patch
$ export DISCORD_BOT_TOKEN="YOUR_BOT_TOKEN"

# เขียนไฟล์ patch แล้ว apply
$ cat > discord.patch.json5 <<'JSON5'
{
  channels: {
    discord: {
      enabled: true,
      token: { source: "env", provider: "default", id: "DISCORD_BOT_TOKEN" },
    },
  },
}
JSON5

$ openclaw config patch --file ./discord.patch.json5 --dry-run   # ดูก่อนว่าจะเปลี่ยนอะไร
$ openclaw config patch --file ./discord.patch.json5            # apply จริง
$ openclaw gateway restart
  • token: { source: "env", ... } = SecretRef ชี้ไปที่ env แทนที่จะเก็บ token ดิบในไฟล์ (จะใส่ token ตรง ๆ ก็ได้ แต่ไม่แนะนำ)
  • ถ้ารันเป็น service ให้เก็บ env ใน ~/.openclaw/.env เพื่อให้ service resolve ได้หลัง restart
  • env DISCORD_BOT_TOKEN ใช้ได้กับ "บัญชี default" เท่านั้น · config ชนะ env
⏱️
ถ้าโดน Discord rate-limit ตอน startup ให้ใส่ channels.discord.applicationId (เลข Application ID จาก Developer Portal) เพื่อให้ข้าม REST lookup ตอนเริ่ม

08 ตั้งทั้ง server เป็น workspace

หลัง DM ใช้ได้แล้ว — เปิดให้ agent ตอบในทุกห้องของ server

ใส่ Server ID + User ID (จากข้อ 05) ลง allowlist ของ guild:

~/.openclaw/openclaw.json — guild workspace
{
  channels: { discord: {
    groupPolicy: "allowlist",
    guilds: {
      YOUR_SERVER_ID: {
        requireMention: false,        // ตอบทุกข้อความ (server ส่วนตัว)
        users: ["YOUR_USER_ID"]        // ใครในกลุ่มสั่งได้
      }
    }
  } }
}
  • groupPolicy: "allowlist" = ค่าปลอดภัยพื้นฐาน · guild ต้อง match ใน guilds
  • requireMention ค่าเริ่มต้น true (ต้อง @mention) — server ส่วนตัวมักตั้ง false ให้ตอบทุกข้อความ
  • ถ้า guild มี block channels = อนุญาตเฉพาะห้องที่ระบุ · ถ้าไม่มี = ทุกห้องในกลุ่มนั้นได้หมด
🤐
กับดักสำคัญที่สุดของ Discord: ในห้องกลุ่ม คำตอบ final ของ agent เป็น private โดยปริยาย — agent ต้องเรียก message tool เองเพื่อโพสต์ให้เห็น

ถ้าเห็น bot "พิมพ์อยู่" + log มี token usage แต่ไม่มีข้อความโผล่ ให้ดู log หา didSendViaMessagingTool: false — แปลว่าโมเดลตอบแบบ private แทนที่จะเรียก tool

แก้ได้ 2 ทาง: (1) ใช้โมเดลที่เรียก tool เก่ง — Codex/openai/gpt-5.5 ทำได้ดี หรือ (2) คืนพฤติกรรมเดิม: ตั้ง messages.groupChat.visibleReplies: "automatic"
🧠
เรื่อง memory: ไฟล์ MEMORY.md โหลดเฉพาะใน DM session เท่านั้น · ห้องกลุ่มไม่ auto-load — ถ้าต้องการ context ร่วมทุกห้อง ให้ใส่ใน AGENTS.md / USER.md (inject ทุก session)

09 🔥 ผูกห้อง Discord เข้ากับ Codex (ACP)

หัวข้อสำหรับสายโค้ดโดยเฉพาะ — เปลี่ยนห้อง Discord ให้เป็น "หน้าต่างคุยกับ Codex" ที่ทำงานในโปรเจกต์จริงต่อเนื่อง

OpenClaw รัน Codex เป็น ACP agent แบบ persistent ที่ผูกกับ working directory ของโปรเจกต์ได้ แล้ว bind เข้ากับห้อง Discord ห้องหนึ่ง — ทุกข้อความในห้องนั้นจะวิ่งเข้า session Codex เดิม ทำให้มันจำ context งานต่อเนื่องเหมือนเปิด Codex ค้างไว้ในห้องแชต

~/.openclaw/openclaw.json — Codex ACP binding
{
  agents: {
    list: [
      {
        id: "codex",
        runtime: {
          type: "acp",
          acp: {
            agent: "codex",
            backend: "acpx",
            mode: "persistent",
            cwd: "/workspace/pedpro"      // โฟลเดอร์โปรเจกต์จริง
          }
        }
      }
    ]
  },
  bindings: [
    {
      type: "acp",
      agentId: "codex",
      match: {
        channel: "discord",
        accountId: "default",
        peer: { kind: "channel", id: "YOUR_CHANNEL_ID" }
      },
      acp: { label: "codex-main" }
    }
  ],
  channels: { discord: { guilds: {
    YOUR_SERVER_ID: { channels: {
      YOUR_CHANNEL_ID: { requireMention: false }
    } }
  } } }
}

หรือสั่งสด ๆ จากในห้องด้วย slash command

Discord — ในห้องที่ต้องการ
/acp spawn codex --bind here   # ผูกห้องนี้กับ Codex session ในที่
/new                          # รีเซ็ต session เดิมในห้องที่ผูกไว้
/unfocus                      # ยกเลิกการผูก thread
🔥
เคสใช้งานจริง: ตั้งห้อง #pedpro-codex ผูกกับ Codex ที่ cwd ชี้ไปโฟลเดอร์ repo → พิมพ์งานจากมือถือ เช่น "เพิ่ม validation ให้ฟอร์ม BotLine ตาม Bruno spec" แล้ว Codex ทำงานในโปรเจกต์จริงให้ ขณะคุณไม่ได้นั่งหน้าจอ

10 Slash · Voice · Streaming

ฟีเจอร์เด่นเฉพาะของ Discord (สรุปสั้น เจาะลึกใน Part 4)

Slash commands

เปิดอัตโนมัติ (commands.native: "auto") คำสั่งเป็น ephemeral (เห็นคนเดียว) โดยปริยาย ตัวอย่าง:

  • /model, /models — เปิด picker เลือก provider/model/runtime แบบ dropdown
  • /focus <target> / /unfocus — ผูก/ปลด thread กับ session
  • /agents — ดู run ที่ทำงานอยู่ + สถานะ binding

Voice channel

Discord — voice
/vc join channel:<voice-channel-id>
/vc status
/vc leave

ต้องให้สิทธิ์ Connect/Speak ในห้องเสียง + เปิด channels.discord.voice.enabled: true · voice message ต้องมี ffmpeg + ffprobe บนเครื่อง

Streaming

channels.discord.streaming ค่าเริ่มต้น off (ต่างจาก Telegram) เพราะการแก้ข้อความ preview บน Discord ชน rate limit ง่าย — เปิดเป็น block หรือ progress ได้ถ้าต้องการเห็นความคืบหน้า
