01 สร้าง bot ใน BotFather

BotFather คือ bot ทางการของ Telegram ที่เอาไว้สร้าง/จัดการ bot ตัวอื่น

1
เปิด Telegram ค้นหา @BotFather (เช็กให้ตรงเป๊ะ มีเครื่องหมายติ๊กถูกสีฟ้า) แล้วกดเริ่มแชต
2
พิมพ์คำสั่ง /newbot
3
ตั้ง display name (ชื่อที่โชว์) เช่น My Pedpro Agent
4
ตั้ง username ที่ลงท้ายด้วย bot หรือ _bot เช่น pedpro_agent_bot (ต้องไม่ซ้ำใคร)
5
BotFather จะส่ง Bot Token กลับมา หน้าตาแบบ 123456789:ABCdefGHI... — คัดลอกเก็บไว้
🔐
Token = รหัสผ่านของ bot ใครได้ไปคุม bot คุณได้ทันที — อย่าเผยแพร่ อย่า commit ขึ้น Git ถ้าหลุดให้ขอใหม่ผ่าน BotFather /mybots → API Token → Revoke

02 ใส่ token ให้ OpenClaw

มี 2 ทาง — เลือกอย่างใดอย่างหนึ่ง ถ้าตั้งทั้งคู่ "config ชนะ env"

ทาง A — แก้ config โดยตรง (แนะนำ ชัดเจน)

เปิดไฟล์ ~/.openclaw/openclaw.json แล้วเพิ่มบล็อก channels.telegram:

~/.openclaw/openclaw.json (JSON5)
{
  channels: {
    telegram: {
      enabled: true,
      botToken: "123456789:ABCdefGHI...",   // token จาก BotFather
      dmPolicy: "pairing",                  // ค่าเริ่มต้น (ดูข้อ 03)
      groups: { "*": { requireMention: true } }
    }
  }
}

ทาง B — สั่งผ่าน CLI

bash — config set
$ openclaw config set channels.telegram.botToken "123456789:ABCdefGHI..."
$ openclaw config set channels.telegram.enabled true

ทาง C — Environment variable

ใช้ได้กับ "บัญชี default" เท่านั้น เหมาะเวลารัน on server ที่ไม่อยากให้ token อยู่ในไฟล์ config:

bash — env fallback
$ export TELEGRAM_BOT_TOKEN="123456789:ABCdefGHI..."
💾
แก้เสร็จแล้ว restart Gateway เสมอ เพื่อโหลด config ใหม่:
bash
$ openclaw gateway restart
ใน log ควรเห็นบรรทัดทำนอง starting provider (@YourBotUsername) = เชื่อมสำเร็จ

03 เข้าใจ dmPolicy ก่อนทักแชต

dmPolicy คุมว่า "ใครส่ง DM สั่ง bot ได้บ้าง" — เข้าใจตรงนี้ = ตั้งความปลอดภัยถูกตั้งแต่ต้น

dmPolicyพฤติกรรมเหมาะกับ
pairing (default)คนใหม่ทักได้ แต่ต้องให้คุณ approve รหัสจับคู่ก่อนเริ่มต้น / ทดสอบ ปลอดภัยดี
allowlistเฉพาะ user ID ที่อยู่ใน allowFrom เท่านั้น (ต้องมีอย่างน้อย 1 ID)bot ส่วนตัวเจ้าของคนเดียว — แนะนำสุด
openใครก็สั่งได้ ต้องตั้ง allowFrom: ["*"]bot สาธารณะที่จำกัด tool แน่นหนาแล้วเท่านั้น
disabledปิด DM ทั้งหมดใช้เฉพาะในกลุ่ม
🚨
อย่าใช้ open + ["*"] มั่ว ๆ! ใครที่เดา username ของ bot เจอจะสั่งงาน agent คุณได้ทันที (และเผา token/เงินค่าโมเดลของคุณ) — bot เจ้าของคนเดียวให้ใช้ allowlist เสมอ
ℹ️
dmPolicy: "allowlist" ที่ allowFrom ว่างเปล่า = บล็อกทุกคน และจะถูก config validation ปฏิเสธ — ต้องใส่ numeric user ID อย่างน้อยหนึ่งตัว (วิธีหา ID อยู่ข้อ 05)

04 Pairing — ทักครั้งแรก

ถ้าใช้ pairing (ค่าเริ่มต้น) นี่คือวิธีให้ตัวเองเข้าใช้ครั้งแรก

1
ให้ Gateway รันอยู่ (เช็กด้วย openclaw gateway status หรือสตาร์ทด้วย openclaw gateway)
2
เปิด Telegram หา bot ของคุณ (ตาม username ที่ตั้งไว้) แล้วกด Start / ทักข้อความอะไรก็ได้
3
ดูรหัสจับคู่ที่ฝั่ง server แล้ว approve:
bash — pairing
$ openclaw pairing list telegram            # ดูคำขอจับคู่ที่ค้างอยู่ + รหัส
$ openclaw pairing approve telegram <CODE>   # อนุมัติด้วยรหัสที่เห็น
4
กลับไปที่ Telegram ทักหา bot อีกครั้ง เช่น "สวัสดี ช่วยอะไรได้บ้าง" — agent จะตอบกลับผ่าน Codex 🎉
⏰
รหัส pairing หมดอายุใน 1 ชั่วโมง ถ้าช้าไปให้ทัก bot ใหม่เพื่อสร้างรหัสใหม่
👑
โบนัส: การ approve pairing ครั้งแรกจะตั้งคุณเป็น commands.ownerAllowFrom โดยอัตโนมัติ (เจ้าของคำสั่งระดับ owner เช่นการอนุมัติ exec) — ดังนั้นเซ็ตอัปเจ้าของคนเดียวก็ใช้งานได้เลยโดยไม่ต้องตั้งซ้ำ

05 หา Telegram user ID ของคุณ

จำเป็นสำหรับ allowFrom — ต้องเป็น ตัวเลขล้วน (ไม่ใช่ @username)

วิธีที่ 1 — ปลอดภัยสุด (ไม่พึ่ง bot คนอื่น)

bash — อ่านจาก log
$ openclaw logs --follow
# แล้วทัก bot ของคุณ → มองหาค่า from.id ใน log นั่นคือ user ID

วิธีที่ 2 — Telegram Bot API ทางการ

bash — getUpdates
$ curl "https://api.telegram.org/bot<bot_token>/getUpdates"
# ทัก bot ก่อนหนึ่งครั้ง แล้วดู "from":{"id": ... } ใน JSON

วิธีที่ 3 — bot ภายนอก (ส่วนตัวน้อยกว่า)

ทักหา @userinfobot หรือ @getidsbot มันจะตอบ user ID ของคุณกลับมา — สะดวกแต่ต้องเชื่อใจ bot นั้น

06 ล็อกให้เป็น "เจ้าของคนเดียว"

นี่คือเซ็ตอัปที่แนะนำสำหรับใช้ส่วนตัว — สิทธิ์ฝังใน config ถาวร ไม่พึ่ง pairing

เปลี่ยนจาก pairing มาเป็น allowlist พร้อมใส่ user ID ของคุณ (จากข้อ 05):

~/.openclaw/openclaw.json — owner-only
{
  channels: {
    telegram: {
      enabled: true,
      botToken: "123456789:ABCdefGHI...",
      dmPolicy: "allowlist",
      allowFrom: ["8734062810"]        // ← user ID ของคุณ (ตัวเลขล้วน)
    }
  },
  commands: {
    ownerAllowFrom: ["telegram:8734062810"]   // ← สิทธิ์ owner เต็ม (มี prefix telegram:)
  }
}
  • allowFrom รับ numeric user ID · prefix telegram: / tg: ใส่ได้ ระบบ normalize ให้
  • commands.ownerAllowFrom = คนที่สั่งคำสั่งระดับ owner ได้ (เช่น approve การรันคำสั่ง) — ใส่แบบมี prefix telegram:<id>
  • เซ็ตแบบนี้ "คุณได้รับสิทธิ์ครั้งเดียว ใช้ได้ทั้ง DM และคำสั่งในกลุ่ม"
🔧
ถ้า config เก่ามี @username ใน allowlist (รูปแบบเลิกใช้แล้ว) รัน openclaw doctor --fix เพื่อแปลงเป็น numeric ID ให้ (ต้องมี bot token เพื่อ resolve)

07 ใช้งานในกลุ่ม (Groups)

ตรงนี้มีกับดักเยอะ — มี 2 ชั้นการควบคุมที่ต้องตั้งให้ครบ

2 ชั้นที่ต้องเข้าใจ

  • กลุ่มไหนได้บ้าง → channels.telegram.groups (ค่าเริ่มต้น groupPolicy: "allowlist" = บล็อกจนกว่าจะเพิ่มกลุ่ม หรือใส่ "*")
  • ใครในกลุ่มสั่งได้ → groupPolicy (open / allowlist ค่าเริ่มต้น / disabled) ร่วมกับ groupAllowFrom (ถ้าไม่ตั้งจะ fallback ไปใช้ allowFrom)

ขั้นตอน

1
เพิ่ม bot เข้ากลุ่ม
2
หา group chat ID — อ่าน chat.id จาก openclaw logs --follow, forward ข้อความกลุ่มไปหา @userinfobot, หรือดูจาก getUpdates
3
ใส่ group ID ลง config (ID supergroup เริ่มด้วย -100)
~/.openclaw/openclaw.json — owner-only group
{
  channels: {
    telegram: {
      enabled: true,
      dmPolicy: "allowlist",
      allowFrom: ["8734062810"],
      groupPolicy: "allowlist",
      groups: {
        "-1001234567890": { requireMention: true }   // ← group chat ID
      }
    }
  }
}

ทดสอบในกลุ่มด้วย @<bot_username> ping — ขณะ requireMention: true ข้อความธรรมดาที่ไม่ mention จะไม่ทำให้ bot ตอบ

📍
กับดักที่สับสนที่สุด: ID กลุ่มติดลบ (-100...) ต้องอยู่ใต้ groups เท่านั้น — ห้ามเอาไปใส่ใน groupAllowFrom (ตรงนั้นเอาไว้ใส่ user ID ของคนในกลุ่มเท่านั้น)

Privacy Mode — ทำไม bot ไม่เห็นข้อความในกลุ่ม

bot ของ Telegram เปิด Privacy Mode เป็นค่าเริ่มต้น ทำให้เห็นเฉพาะข้อความที่ mention ถึงมัน ถ้าอยากให้ bot เห็นทุกข้อความ (เช่น requireMention: false) ต้องเลือกทำอย่างใดอย่างหนึ่ง:

  • ปิด privacy ที่ BotFather: /setprivacy → Disable
  • หรือทำให้ bot เป็น admin ของกลุ่ม
  • สำคัญ: หลังสลับ privacy mode ต้อง ลบ bot ออกแล้วเพิ่มกลับเข้ากลุ่ม เพื่อให้ Telegram ใช้ค่าใหม่
🧵
ขั้นสูง: Forum/Topics รองรับการแยก agent ต่อ topic ได้ด้วย groups.<chatId>.topics.<threadId>.agentId — เช่น topic dev ส่งไป agent ตัวหนึ่ง, topic code review ส่งไปอีกตัว แต่ละ topic มี session/memory แยกกัน

08 Streaming + การโชว์งานของ Codex

เกี่ยวกับ Codex โดยตรง — คุมว่าจะเห็น "ความคืบหน้า" ของ agent แค่ไหนระหว่างทำงาน

OpenClaw stream คำตอบแบบ real-time ได้ คุมด้วย channels.telegram.streaming ซึ่งมี 4 โหมด เวลา Codex ทำงาน (รันคำสั่ง อ่านไฟล์ วางแผน) มันจะมี "tool-progress preview" — รวมถึงข้อความ preamble/commentary ของ Codex app-server ที่โชว์ขึ้นมาเป็นบรรทัดสถานะสั้น ๆ

modeพฤติกรรม
partial (default)แก้ข้อความเดิมไปเรื่อย ๆ โชว์คำตอบทยอยขึ้น + โชว์ความคืบหน้า tool
progressโชว์สถานะ tool แยก แล้วส่งคำตอบสุดท้ายเป็นข้อความใหม่ (ไม่แก้ทับ)
blockstream เป็นบล็อก
offส่งเฉพาะคำตอบสุดท้าย เงียบ tool chatter ทั้งหมด

ถ้าอยากให้คำตอบทยอยขึ้นแต่ ซ่อนบรรทัดความคืบหน้า tool:

~/.openclaw/openclaw.json
{
  channels: { telegram: {
    streaming: {
      mode: "partial",
      preview: { toolProgress: false }
    }
  } }
}
🔧
ถ้า config เก่าใช้ streamMode (รูปแบบเลิกใช้) รัน openclaw doctor --fix เพื่อย้ายมาเป็น streaming.mode ให้

09 สั่งจาก CLI + เมนูคำสั่ง

นอกจากแชต ยังสั่ง OpenClaw ให้ส่งข้อความ/โพลเข้า Telegram ผ่าน command line ได้

ส่งข้อความ / โพล

bash — message send / poll
# ส่งหา user ID, @username หรือ topic ในกลุ่ม
$ openclaw message send --channel telegram --target 123456789 --message "สวัสดี"

# สร้างโพลในกลุ่ม
$ openclaw message poll --channel telegram --target 123456789 \
    --poll-question "deploy เลยไหม?" --poll-option "เอา" --poll-option "รอ"

เพิ่มเมนูคำสั่งของตัวเอง

เพิ่มรายการในเมนู / ของ Telegram (เป็นแค่รายการเมนู ไม่ได้สร้าง logic ให้อัตโนมัติ):

~/.openclaw/openclaw.json
{
  channels: { telegram: {
    customCommands: [
      { command: "backup", description: "สำรองด้วย git" },
      { command: "generate", description: "สร้างรูป" }
    ]
  } }
}

กติกา: ชื่อใช้ a-z 0-9 _ ยาว 1–32 ตัว · override คำสั่ง native ไม่ได้ · ที่ซ้ำจะถูกข้าม
