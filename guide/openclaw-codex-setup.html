ติดตั้ง OpenClaw + Codex
แบบล้างเริ่มใหม่

OpenClaw ไม่ใช่ AI ตัวใหม่ที่ต้องเรียนใหม่หมด — มันคือ gateway ตัวกลาง ที่รันค้างบน server ตัวเดียว เชื่อมแอปแชต (Telegram) เข้ากับ coding agent อย่าง Codex. Codex คือ คนทำงาน, OpenClaw คือ ประตูหน้า, ส่วนคุณคือ manager ที่กด approve.

อ่านก่อนลงบน server บริษัท — สำคัญกว่าทุก step

OpenClaw แทบจะเหมือนให้ LLM มีสิทธิ์ระดับ sudo บนเครื่องที่มันรัน ถ้าตั้งค่าผิดคือช่องโหว่ใหญ่ และการป้อน proprietary code เข้าไป = ส่งข้อมูลบริษัทออกไปข้างนอก

  • Isolate — รันบน container / VM แยก ไม่ใช่เครื่อง prod ที่ใช้งานจริงอยู่
  • Allowlist — ล็อกให้คุยกับ bot ได้เฉพาะ Telegram id ของคุณคนเดียว
  • อย่าชี้ไปที่ production secrets / repo ที่ห้ามหลุด
  • ระวัง skill จาก ClawHub — มีประวัติเคส supply-chain (ที่คุณเคยศึกษาจาก Arise #7)
  • คุยกับทีมให้ชัดเรื่อง scope ก่อนเริ่ม

เช็กให้พร้อมก่อน อย่าเพิ่งแตะ install จนกว่าจะตอบ 5 ข้อนี้ได้

terminal
node -v   # ต้องการ Node 24 (แนะนำ) หรือ 22.19+ — installer ลงให้เองได้

ถอนของเดิมให้เกลี้ยงก่อนลงใหม่ จะได้ไม่มี config เก่าค้างมากวน

terminal
openclaw uninstall --all --yes --non-interactive
terminal · linux systemd
systemctl --user disable --now openclaw-gateway.service
rm -f ~/.config/systemd/user/openclaw-gateway.service
systemctl --user daemon-reload
terminal
rm -rf ~/.openclaw
npm rm -g openclaw          # หรือ pnpm remove -g / bun remove -g
which openclaw              # ควรไม่เจออะไรเลย = สะอาดแล้ว

installer script จะ detect OS, ลง Node ให้ถ้าจำเป็น, ลง OpenClaw แล้วเปิด onboarding ให้

terminal · ubuntu
curl -fsSL https://openclaw.ai/install.sh | bash
อยากลงก่อนแล้วค่อย onboard ทีหลัง? เติม bash -s -- --no-onboard ต่อท้าย
terminal
openclaw --version
openclaw doctor             # เช็กปัญหา config — ทำให้เขียว/ผ่านก่อนไปต่อ
openclaw not found? เพิ่ม global bin เข้า PATH: export PATH="$(npm prefix -g)/bin:$PATH" ใน ~/.bashrc แล้วเปิด terminal ใหม่

คุณมี GPT Plus → ใช้ route แบบ subscription / OAuth (ไม่ใช่ API key) คือ prefix openai-codex/* ซึ่งเป็นตัวเลือกแรกที่ดีที่สุดสำหรับ subscription setup

terminal
openclaw onboard --auth-choice openai-codex
terminal · headless server
openclaw models auth login --provider openai-codex --device-code
terminal
openclaw config set agents.defaults.model.primary openai-codex/gpt-5.5
openclaw models list --provider openai-codex
ตรวจสอบ: Codex เข้าถึงได้แค่ไหนขึ้นกับ tier ของ ChatGPT plan ถ้า Plus ติด quota หรือเข้าไม่ได้ ค่อยพิจารณา route แบบ API key (openai/* + OPENAI_API_KEY) แทน

Telegram เป็น channel ที่ต่อเร็วสุด นี่คือ "ประตู" ที่คุณจะสั่งงาน Codex จากมือถือ

~/.openclaw/openclaw.json · รูปแบบตัวอย่าง
{
  channels: {
    telegram: { allowFrom: ["<your-telegram-user-id>"] }
  }
}
key ที่ถูกต้องให้ดูตามที่ onboarding ถาม หรือหน้า docs.openclaw.ai/channels/telegram — อันนี้เป็นแพตเทิร์น allowlist เพื่อให้เห็นภาพ

ทำให้ gateway รันค้างถาวร (systemd user service) แล้วลองยิงจริง

terminal
openclaw onboard --install-daemon
openclaw gateway status
terminal · เครื่องคุณ (ไม่ใช่ server)
ssh -L 18789:127.0.0.1:18789 user@your-server
# แล้วเปิด http://127.0.0.1:18789 บน browser เครื่องคุณ — หรือใช้ Tailscale

มันคือ pattern เดิมที่คุณทำมาแล้ว — "parallelize agents, serialize human judgment" แค่เปลี่ยน worker เป็น Codex และประตูหน้าเป็น OpenClaw ส่วนคุณคือ manager ที่กด approve. ไม่ต้องเข้าใจทุก layer วันนี้ ค่อยขยาย scope ทีละขั้นแบบ Stage 0 ที่วางไว้

1
เริ่มจากงาน read-only ก่อนtext จากมือถือถาม Codex เรื่อง codebase / ให้สรุป PR — ยังไม่ให้แตะ git ดูว่า routing + Telegram + approve flow เวิร์ก
2
ต่อ Plan Review Gate ของคุณเข้ามาสั่งงาน → Codex ทำใน worktree → ส่ง diff + summary กลับ Telegram → กด "Open diff" แล้วค่อย /approve
3
งานประจำผ่าน cron / heartbeatdaily-standup, ticket-to-vault จาก YouTrack ให้รันเองแล้วเด้งเข้า Telegram
4
xFish workflowYouTrack ticket → ให้ Codex ร่าง plan ตาม CLAUDE.md → รอคุณ approve ก่อนลงมือ

openclaw gateway status      # สถานะ gateway
openclaw dashboard           # เปิด Control UI

openclaw update              # อัปเดตเวอร์ชัน
