Quickstart

เริ่มเล่นใน 30 วินาที

ไม่ต้องมี YouTrack/Telegram/API key — demo mode รันลูปทั้งวงด้วย local fakes

terminal
unzip openclaw-xfish-kit.zip && cd kit

just demo      # spawn 2 runs ขนาน → goal_met → เข้าคิว merge gate → metrics
just test      # 18 tests (gate routing / YouTrack race / Telegram dispatch)
just lint      # ruff

# ใช้จริง: เปิด .env.example → .env, implement seam ใน integration.py
docker build -t openclaw . && docker run --env-file .env -v $(pwd):/app/xfish openclaw

จุดเริ่มแก้: openclaw/integration.py (seam ที่ต้อง wire) และ CONTRIBUTING.md (วินัยการแก้ระบบ: แก้ instruction → รัน eval → merge)

ไม่อยากแตก zip? เปิด openclaw-source-book.html — มีซอร์สครบทั้ง 45 ไฟล์ กดปุ่ม Copy ทีละไฟล์แล้ววางตาม path ได้เลย

The One Rule

กฎเดียวที่อยู่เหนือทุก test

ขนาน agent ให้สุด
— serialize วิจารณญาณเสมอ

ระบบทั้งหมดนี้กัน failure ด้านเทคนิคได้: type พังหยุดเอง, push เองไม่ได้, งานแย่งกันไม่ได้, การตัดสินใจที่ gate มีเทสต์คุม. แต่ความล้มเหลวที่ร้ายที่สุดไม่ใช่เทคนิค — มันคือมนุษย์กด approve โดยไม่เปิดดู (บทเรียน Starbucks "Tank Day": ผู้บริหาร 7 คนเซ็นผ่านงานที่ไม่มีใครเปิดไฟล์แนบ)

การแก้ใด ๆ ที่ทำให้ "อนุมัติง่ายขึ้นโดยไม่ต้องดู" — auto-approve, batching, ตัดเงื่อนไขเหตุผล — คือ regression แม้ทุก test จะเขียว

▸ metric ที่ต้องเฝ้าที่สุด: blind_approve_rate — ตราบใดที่มันต่ำ แปลว่าวิจารณญาณยังตื่นอยู่จริง
