# OpenClaw × Pedpro — Agentic Workflow Kit

ไฟล์จริงทั้งหมดจากซีรีส์ handbook 4 พาร์ท วางลง repo ได้เลย
(`.claude/` ลงที่ root ของ Pedpro, `openclaw/` เป็น middleware แยก).

ทุกไฟล์ใน `openclaw/` เป็น **reference scaffold** — เชื่อมกับระบบจริงผ่าน
`openclaw/integration.py` (จุดที่ raise NotImplementedError คือสิ่งที่ต้อง wire เอง).
ส่วน `.claude/` ใช้งานได้ทันทีหลังปรับชื่อ script/path ให้ตรง repo.

แก่นเดียวที่ร้อยทั้งซีรีส์: **ขนาน agent ให้สุด, serialize วิจารณญาณเสมอ.**

## ซีรีส์เต็ม

| # | เนื้อหา | รูปแบบ |
|---|---|---|
| Part 1 | แนวคิด prompt → agent (context, reasoning, vocab, HITL) | handbook |
| Part 2 | หนึ่งลูปปิดตัวเองได้ (skills/hooks/goal/progress/gate) | handbook → ไฟล์ใน kit นี้ |
| Part 3 | N ลูปขนานปลอดภัย (worktrees/supervisor/fan-in) | handbook → ไฟล์ใน kit นี้ |
| Part 4 | ลูปที่วัด+พัฒนาตัวเอง (trace/metrics/eval/judge/alerts) | handbook → ไฟล์ใน kit นี้ |
| **kit** | ไฟล์จริงรันได้ + live + Docker + 18 tests + CI | **โฟลเดอร์นี้** |

## รันได้ทันที (demo mode)

ไม่ต้องมี YouTrack/Telegram/API key — `OPENCLAW_MODE=demo` จะสลับทุก seam เป็น
local fakes (`openclaw/integration_demo.py`) ให้ลูปทั้งวงรันจริง:

```
just demo      # spawn 2 runs ขนาน -> loop -> goal_met -> เข้าคิว merge gate -> metrics
just eval      # รัน regression suite (golden.jsonl) ด้วย demo wiring
just test      # smoke test (pytest)
just verify    # ตรวจ bash/json/python syntax
```

ตัวอย่างผล `just demo` (ของจริงจากการรัน):

```
== driving loops ==
   [telegram] [PED-1423 · a91444] approve merge?
              5 files changed, 142 insertions(+), 18 deletions(-) | checks: {...pass...}
              reply: /approve a91444 <reason>   or   /reject a91444 <reason>
   [telegram] [PED-1480 · 17b9b3] approve merge?  ...
== metrics rollup ==
   runs 2 | success_rate 1.0 | intervention_rate 0.0 | median_iters 2 | usd_per_issue 0.84
```

ทุก run ขนานกัน แต่แต่ละ merge มาเป็นคำขออนุมัติ**แยกตัว มี tag issue+id** เข้าช่อง
เดียว — นี่คือ "ขนาน agent, serialize วิจารณญาณ" ที่เห็นเป็นรูปธรรม.
เมื่อพร้อมใช้จริง ลบ env `OPENCLAW_MODE` แล้ว implement seam ใน `integration.py`.

## ใช้จริง (live mode)

`OPENCLAW_MODE=live` ใช้ `openclaw/integration_live.py` ที่ wire ของจริง:

| seam | ของจริงที่ต่อ |
|---|---|
| YouTrack | REST `/api/issues` + `/api/commands` (claim = optimistic-verify เพราะ YouTrack ไม่มี CAS) |
| Telegram | Bot API `sendMessage` + inline button "Open diff" ผ่าน tracker |
| Anthropic | `AsyncAnthropic().messages.create` (judge) |
| Claude Code | `claude -p --output-format json --max-turns N --model ... --permission-mode dontAsk` → parse `total_cost_usd`/`usage`, แล้ว**รัน acceptance เองเพื่อตัดสิน goal_met** (ไม่เชื่อคำโมเดล) |

ตั้งค่าใน `.env` (ดู `.env.example`): `YOUTRACK_BASE_URL/TOKEN`, `TELEGRAM_BOT_TOKEN/CHAT_ID`,
`ANTHROPIC_API_KEY`. ติดตั้ง deps: `pip install -r openclaw/requirements.txt`.

> ⚠️ verify endpoint shape กับ YouTrack/Telegram เวอร์ชันคุณก่อน — โค้ดเป็น reference.
> `claude -p` headless field `total_cost_usd`/`usage` อิง docs ปัจจุบัน (code.claude.com/docs/en/headless).

### Docker

`Dockerfile` ทำ image ของ middleware (python + node + claude-code + pnpm + git + jq):

```
docker build -t openclaw .
docker run --rm --env-file .env -v /path/to/pedpro:/app/pedpro openclaw
```

mount repo Pedpro เข้าไป; worktrees จะถูกสร้างใต้ `OPENCLAW_WORKTREE_BASE`.

### ปิดลูปอนุมัติ (inbound)

`telegram_service.py` คือขาเข้าที่ขาดไป — long-poll `getUpdates` แล้ว route:

```
ส่งคำขอ (supervisor) ──▶ Telegram  [PED-1423 · 8994d7] + ปุ่ม [Open diff]
   ผู้ใช้แตะ [Open diff] ─▶ callback "open:<rid>" ─▶ gate.mark_diff_opened(rid,user)
   ผู้ใช้พิมพ์ /approve 8994d7 reviewed... ─▶ on_reply ถอด tag ─▶ gate.handle_decision
        ตรวจ: เปิด diff แล้ว? + มีเหตุผล? ──▶ ผ่าน ─▶ execute(push/deploy)
```

`run.py` รัน supervisor + listener **ในโปรเซสเดียว** เพื่อแชร์คิวอนุมัติในเมมโมรี
(หลายโปรเซส → ย้าย `PENDING`/`_OPENED` ไป Redis/SQLite). พฤติกรรม anti-rubber-stamp
มีเทสต์คุม: `pytest openclaw/tests` (**18 ผ่าน**, mock-free/network-free):
gate routing (approve โดยไม่เปิด diff/ไม่มีเหตุผล/ไม่มี tag ถูก block, คำขอค้างคิว),
YouTrack optimistic-verify claim (จำลอง race ด้วย httpx mock — ใครชนะ re-read คนนั้นได้งาน),
และ telegram dispatch (จำลอง payload callback/message จริง). `ci.yml` รัน `ruff check` + `pytest` ให้ทุก push/PR (`just lint` / `just test` รันเองก็ได้).

## ข้อควรระวัง

- **ทดสอบ `exit 2` จริงก่อนพึ่งเป็นด่านความปลอดภัย** — แก้ไฟล์ให้ type พังแล้วดูว่า
  typecheck hook block จริง; ลองสั่ง `git push` ดูว่า git-gate block จริง.
- **eval รันโมเดลจริง = มีต้นทุน** — รันเมื่อแตะ `.claude/`/`openclaw/` หรือ nightly
  ไม่ใช่ทุก commit. golden set เล็กแต่ครอบ failure mode สำคัญ.
- **judge มี bias + drift** — calibrate กับ human spot-check เป็นระยะ,
  ห้าม judge อนุมัติ irreversible action แทนคน.
- **`blind_approve_rate` คือ metric ที่ต้องเฝ้าที่สุด** — ถ้ามันไต่ขึ้น แปลว่า gate
  กำลังกลายเป็นตรายาง (โรค Starbucks "Tank Day"). gate มีไว้ให้ "ดูจริง" ไม่ใช่ "กดผ่าน".

# Contributing / changing the system

Two kinds of change live in this repo. Keep them separate.

## 1. Product change (a Pedpro feature)
The agent loop handles these. You write the YouTrack issue + acceptance; the
loop implements until `goal.json` acceptance is green, then routes a merge to
the Telegram gate. You review the diff and `/approve <tag> <reason>`.

## 2. System change (the agent itself)
Editing **CLAUDE.md, a skill, a hook, a prompt, or anything in `openclaw/`**
changes how every future run behaves. Treat instructions like code:

```
1. make the change
2. just lint           # ruff
3. just test           # 18 mock/network-free tests
4. just eval           # golden.jsonl run through the whole loop (costs tokens)
5. open a PR           # ci.yml gates lint+test; agent-eval.yml gates pass@1
```

**Never change an instruction without running eval.** A prompt tweak that helps
one case often regresses another; the golden set is how you find out before
shipping, not after.

### When you find a new failure mode in production
From metrics/traces (Part 4 §06), if you spot a recurring failure (a guessed
field, a skipped test, a runaway iteration count):

1. add a **golden task that reproduces it** to `openclaw/golden.jsonl` first
2. then fix the instruction/skill/hook
3. confirm the new task passes AND nothing else regressed

That is TDD applied to the agent system — the same discipline `pedpro-tdd`
asks of product code.

## Local dev
- `just demo`   — run the whole loop with local fakes (no creds)
- `just test`   — pytest
- `just lint`   — ruff
- `mypy openclaw` — optional, lenient type check (config in pyproject.toml)
- pre-commit: `pip install pre-commit && pre-commit install` (runs ruff + hook syntax on commit)

## The one rule that matters most
Parallelize the agents, serialize the judgment. Any change that makes approval
easier to give without looking (auto-approve, batching, dropping the reason
requirement) is a regression even if every test passes — watch
`blind_approve_rate` (Part 4 §07).

# Pedpro

Pedpro is a Thai-language fantasy gaming portal.

## Stack (facts)
- Vue 3.5+ Composition API, Nuxt 4 (flat root, not app/ layout).
- TypeScript strict mode. Tailwind + Naive UI. Pinia for state.
- Path aliases ~/ and @/ resolve to the project root.
- Page-local components live in pages/<feature>/components/ and need explicit imports.
- Bruno .bru specs in the sibling repo are the contract source of truth.

## Conventions this repo follows
- API field names come from .bru files, not from memory. The bruno-contract
  skill resolves them.
- No custom CSS and no arbitrary Tailwind values; only base utility classes.
- No raw HTML <form> elements; forms use Naive UI components.
- Deploys go to GCP Cloud Run via Dockerfile + cloudbuild.yaml.

## What stays out of scope for the agent
- git push, git reset --hard, and gcloud run deploy are human-approved actions.

"""End-to-end demo of the OpenClaw loop with local fakes. No external deps.

Run from the repo root:   OPENCLAW_MODE=demo python3 openclaw/demo.py
It drives two parallel runs through spawn -> loop -> goal_met -> merge gate,
writes real traces, then rolls them up with the real metrics module.
"""
import os
import sys
import asyncio

os.environ.setdefault("OPENCLAW_MODE", "demo")
sys.path.insert(0, os.path.dirname(__file__))   # make flat imports resolve

import types
import supervisor          # noqa: E402  (imports after env + path setup)
import metrics             # noqa: E402

TRACES = os.path.join(os.path.dirname(__file__), "traces.jsonl")


def issue(key):
    return types.SimpleNamespace(key=key, id=key)


async def main():
    open(TRACES, "w").close()                      # fresh trace file
    issues = [issue("PED-1423"), issue("PED-1480")]
    print("== spawning parallel runs ==")
    runs = [supervisor.spawn_run(i) for i in issues]
    sem = asyncio.Semaphore(len(runs))
    for _ in runs:
        await sem.acquire()
    active = {}
    print("\n== driving loops ==")
    await asyncio.gather(*(supervisor.drive(r, sem, active) for r in runs))

    print("\n== metrics rollup (real metrics.py) ==")
    for k, v in metrics.rollup(TRACES).items():
        print(f"   {k:18} {v}")
    print("\n== cost by model ==")
    for m, agg in metrics.cost_by_model(TRACES).items():
        print(f"   {m:8} usd={agg['usd']:.2f}  iters={agg['iters']}")


if __name__ == "__main__":
    asyncio.run(main())
